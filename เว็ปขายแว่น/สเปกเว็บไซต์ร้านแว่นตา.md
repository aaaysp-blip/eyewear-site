# สเปกเว็บไซต์ร้านแว่นตา

## ภาพรวม
เว็บไซต์ขายแว่นตาออนไลน์ งบประมาณต่ำสุด/ฟรี เริ่มจากกรอบแว่น+แว่นกันแดด+อุปกรณ์เสริมก่อน (แว่นสายตาเป็นเฟสถัดไป)

## ดีไซน์
- สไตล์มินิมอล เรียบง่าย พื้นหลังโทนกลาง ไม่ใช้สีสันจัดจ้าน (เพื่อให้แว่นเลนส์ใสเป็นจุดเด่น)

---

## 1. หน้าร้าน (ฝั่งลูกค้า)

### หมวดหมู่สินค้า (แท็บ)
1. สินค้าเข้าใหม่ (เรียงตามวันที่ลงล่าสุด ขึ้นก่อน มีป้าย "ใหม่" ถ้าลงมาไม่เกิน 3 วัน)
2. แว่นกันแดด
3. กรอบแว่นตา
4. อุปกรณ์เสริม (ผ้าเช็ดแว่น, ซองแว่นตา, กล่องแว่นตา)

### ข้อมูลต่อสินค้า
- รหัสสินค้า (เช่น C1, C2, C3...)
- ชื่อสินค้า
- แบรนด์
- สี (เลือกได้จาก dropdown)
- ราคา
- จำนวนสต็อกคงเหลือ (ถ้า 0 = ปุ่มใส่ตะกร้อถูกปิด ขึ้น "หมดสต็อก")
- ขนาด: หน้าแว่นกว้าง, เลนส์กว้าง, เลนส์สูง (หน่วย มม.) — **แสดงให้ลูกค้าเห็นด้วย** ไม่ใช่แค่หลังบ้าน

### ตัวกรอง/ค้นหา
- ปุ่ม "ตัวกรอง" มุมขวาบน กดแล้วแผงเลื่อนออกมาด้านขวา (มีปุ่มปิด)
- ค้นหาด้วยข้อความ: รหัส, ชื่อ, แบรนด์
- กรองตามแบรนด์ (dropdown)
- กรองตามช่วงขนาด (ต่ำสุด-สูงสุด): หน้าแว่นกว้าง, เลนส์กว้าง, เลนส์สูง

### การแบ่งหน้า
- 12 ชิ้นต่อหน้า, จัดเรียง 4 ชิ้นต่อแถว
- รองรับสินค้า 500+ รายการ

### การ์ดสินค้า → Popup
กดที่การ์ดสินค้าแล้วเด้ง popup (ไม่ใช่หน้าแยก) แสดง:
- รูปสินค้าขนาดใหญ่ขึ้น
- รหัส, ชื่อ, ขนาด
- เลือกสี (ถ้ามีหลายสี)
- ปรับจำนวน (+/- หรือพิมพ์เอง)
- ราคารวม
- ปุ่ม "ใส่ตะกร้า"

### ตะกร้า + เช็คเอาท์ (ขั้นตอน)
1. **ตะกร้า** — สรุปรายการ + ยอดรวม กด "สร้าง QR พร้อมเพย์"
2. **ชำระเงิน** — แสดง QR PromptPay ตามยอดออเดอร์ ลูกค้าสแกนจ่ายเอง (ไม่มีระบบเช็คยอดอัตโนมัติ แอดมินเช็คสลิปเอง — เลือกแบบนี้เพราะไม่มีค่าธรรมเนียม)
3. **ที่อยู่จัดส่ง** — ฟอร์มกรอก:
   - ชื่อ-นามสกุลผู้รับ
   - เบอร์โทร
   - LINE ID (ช่องทางสำรอง เผื่อโทรติดต่อไม่ได้)
   - ที่อยู่ (บ้านเลขที่ ถนน ซอย)
   - แขวง/ตำบล
   - เขต/อำเภอ
   - จังหวัด
   - รหัสไปรษณีย์
4. **ยืนยันคำสั่งซื้อ** — ส่งออเดอร์เข้าระบบหลังบ้าน

## 2. หน้าแอดมิน (หลังบ้าน)

### แดชบอร์ดสรุป
- ยอดขายเดือนนี้
- จำนวนออเดอร์ที่รอดำเนินการ
- สินค้าขายดี

### กล่องเตือนสต็อกใกล้หมด
- แสดงอัตโนมัติเมื่อสินค้าเหลือน้อย (ตั้งเกณฑ์ได้ เช่น ≤2 ชิ้น) หรือหมดสต็อก

### ฟอร์มลงสินค้าใหม่
ช่องกรอก: รหัส, ชื่อสินค้า, แบรนด์, สี, ราคา, จำนวนสต็อก, หน้าแว่นกว้าง, เลนส์กว้าง, เลนส์สูง

**อัปโหลดรูปภาพ:**
- ลากไฟล์วาง (drag & drop) หรือคลิกเลือกไฟล์ รองรับหลายรูป
- **ฟีเจอร์ AI แยกสีอัตโนมัติ:** ถ้าแอดมินอัปโหลดรูปที่มีสินค้าหลายสีอยู่ในรูปเดียวกัน ระบบต้อง:
  1. ตรวจจับตำแหน่งแต่ละชิ้นในรูป
  2. ตัดแยกเป็นรูปย่อยอัตโนมัติ (1 ชิ้น = 1 สี)
  3. เดาชื่อสีให้เบื้องต้น
  4. **ให้แอดมินยืนยัน/แก้ไขชื่อสีและจำนวนสต็อกก่อนกดบันทึกทุกครั้ง** (ห้ามบันทึกอัตโนมัติโดยไม่ผ่านการยืนยัน เพราะ AI เดาสีผิดได้)

### จัดการออเดอร์
สถานะออเดอร์ไหลตามลำดับนี้ (มีปุ่มกดเปลี่ยนสถานะถัดไป):
1. **รอตรวจสลิป** — แอดมินเช็คสลิปโอนเงินที่ลูกค้าส่งมา กดยืนยัน
2. **รอยืนยันเบอร์โทร** — แอดมินโทรหาลูกค้าเพื่อเช็คเบอร์โทร/ที่อยู่ถูกต้อง (กันกรอกเบอร์ผิด) ถ้าโทรไม่ติดให้ใช้ LINE ID ที่กรอกไว้ติดต่อแทน กดยืนยันเมื่อเช็คแล้ว
3. **แพ็คแล้ว**
4. **จัดส่งแล้ว**

### จัดการสต็อก
ตารางแสดงสินค้าทั้งหมดพร้อมช่องแก้ไขจำนวนสต็อกได้ทันที

### CRM ลูกค้า (แบบไม่มีระบบสมาชิก/login)
- **ไม่ทำระบบสมัครสมาชิก/เข้าสู่ระบบ** ในเฟสแรก (จะเพิ่มทีหลังถ้าต้องการ เช่น สะสมแต้ม/ส่วนลดสมาชิก)
- ใช้ **เบอร์โทรเป็นตัวอ้างอิงลูกค้า** — ทุกครั้งที่ลูกค้ากรอกข้อมูลตอนเช็คเอาท์ ระบบบันทึกลง CRM หลังบ้านอัตโนมัติโดยลูกค้าไม่ต้องสมัครสมาชิก
- แอดมินดูประวัติการซื้อของลูกค้าแต่ละเบอร์ได้จากหลังบ้าน

---

## เฟสถัดไป (ยังไม่ทำตอนนี้ แต่วางแผนไว้)
1. **แว่นสายตา (ค่าสายตา)** — เพิ่มตัวเลือกค่าสายตาสำหรับสินค้าประเภทแว่นสายตา ซึ่งซับซ้อนกว่าเพราะบางครั้งลูกค้าเขียนค่าสายตาใส่กระดาษแล้วถ่ายรูปส่งมา
2. **อ่านภาพลูกค้าที่ขีดจำนวน** — ระบบอ่านภาพที่ลูกค้าส่งมา (เขียนรหัสสินค้า+จำนวนที่ต้องการ) แล้วเพิ่มลงตะกร้าอัตโนมัติ
3. **ระบบสมาชิกเต็มรูปแบบ** — สมัครสมาชิก, สะสมแต้ม, ส่วนลดสมาชิก, บันทึกที่อยู่ไว้ใช้ซ้ำ

---

## เทคนิคการนำไปใช้งานจริง (Deployment)
1. เขียนโค้ด frontend (หน้าร้าน+แอดมิน) + backend (เก็บสต็อก/ออเดอร์/ลูกค้าในฐานข้อมูลจริง)
2. ฐานข้อมูล: แนะนำ Supabase หรือ Firebase (มีแพ็กเกจฟรีสำหรับร้านขนาดเล็ก)
3. Hosting: แนะนำ Vercel หรือ Netlify (ฟรีสำหรับการใช้งานทั่วไป)
4. โดเมน: ซื้อเองปีละประมาณ 300-500 บาท (ไม่บังคับ ใช้โดเมนฟรีของ hosting ไปก่อนได้)
5. QR PromptPay: ใช้ไลบรารีสร้าง QR ตามยอดเงิน (ฟรี ไม่ต้องขอ API พิเศษ)
